Enable services

Each service runs in the L1 VM or its own L2 container — an isolation choice. Defaults isolate anything holding credentials or inspecting traffic.

LiteLLMMVP
LLM proxy — agents use virtual keys; real API creds never exposed.
PhoenixMVP
LLM/agent observability — traces, evals, prompt inspection.
VictoriaMetrics + Loki + GrafanaMVP
System metrics + log aggregation (memory-efficient).
SeaweedFSCore
Shared storage across host ↔ containers and container ↔ container.
mitmproxy + ZeekCore
Network inspection / proxy / traffic capture for agent egress.
